How it's built
CLI Layer
Cobra Commands

Four subcommands — migrate, status, rollback, unlock. Global flags: --verbose, --dry-run, --dir. PersistentPreRunE wires config and driver before every command.

Core Engine
Loader + Runner

Loader parses filenames, sorts by version, validates no duplicates, reads Cypher statements. Runner detects schema vs data migrations and routes to separate transaction strategies automatically.

Transaction Safety
Dual-Path Execution

Neo4j forbids mixing DDL and data writes in one transaction. The runner detects CREATE/DROP INDEX keywords and uses per-statement transactions for schema migrations, single atomic transactions for data.

History Tracking
Graph-Native State

Applied migrations are stored as (:SchemaMigration) nodes in Neo4j itself — version, name, timestamp, MD5 checksum. A (:MigrationLock) node prevents concurrent runs.

Migration history
Version Name Type Description
0001 add_genre_nodes Data Creates Genre nodes, wires movies via IN_GENRE relationships. Graph-idiomatic — genres as traversable nodes, not properties.
0002 add_unique_constraints Schema Adds FULLTEXT indexes on Movie (title + tagline) and Person (name + bio). Range indexes on budget and revenue.
0003 add_imdb_rating_band Data + Schema Computes imdbRatingBand property (low/mid/high) on all 28k Movie nodes. Adds range index for fast faceted filtering.
0004 add_release_decade Data + Schema Computes releaseDecade string (e.g. "1990s") from integer year. Indexes it for decade-level analytics grouping.
0005 add_actor_director_labels Data + Schema Propagates Actor, Director, Writer, Producer secondary labels via relationship traversal. Adds label-specific indexes.
0006 rename_acted_in_to_performed_in Data Renames ACTED_IN → PERFORMED_IN across 166k relationships by creating new ones, copying properties, deleting old. Directly mirrors internship work.
CI/CD Pipeline — GitHub Actions
1
Lint & Unit Tests
go vet, gofmt check, and 9 unit tests on the loader package — no Neo4j required. Fast feedback on every push.
2
Download Baseline Dump
Downloads recommendations-50.zip from a pinned GitHub Release asset. Unzips and renames to neo4j.dump for neo4j-admin compatibility.
3
Load Dump into Docker Volume
Runs neo4j-admin database load in a throwaway container before Neo4j starts — avoids the stop/start container problem entirely.
4
Pre-Migration Snapshot
Captures node counts by label, relationship counts by type, active indexes and constraints, and applied migration history to a structured text file.
5
Run All 6 Migrations
Executes graphmigrate migrate --verbose. Each migration runs in the correct transaction mode — schema DDL in separate transactions, data changes atomically.
6
Post-Migration Snapshot + Export
Captures the same state metrics after migration. Exports the full migrated database as a Cypher dump file.
7
Generate Audit Report
Produces a full plain-text audit report: execution log, status table, state diff (node/relationship/constraint changes), and artifact inventory.
8
Store Artifacts (90 days)
Uploads migration report, baseline dump zip, post-migration Cypher export, pre/post snapshots, and raw logs as GitHub Actions artifacts.
